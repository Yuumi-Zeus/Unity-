# 状态模式

## 简介

状态模式可以称作有限状态机

有限状态机的要点：一个角色（实体）能够拥有 ***有限*** 个数的状态，并且当前只有可能处在 ***一个状态*** ，不存在 ***叠加*** 的状态

## 基础原型

1. 状态拥有者 `Context`
   1. 当前状态
   1. 设置状态（切换）方法
   1. 其中包含执行方法，不同状态的共有方法
2. 状态接口 `IState`
   1. 状态拥有者（控制）属性对象
   1. 接口中定义每个状态可以进行的行为方法
3. 状态抽象类 `AbstractState`
   1. 构造函数设置状态拥有者
   2. 继承接口方法，设计为抽象方法

4. 具体状态类 `ConcreteState`
   1. 继承状态抽象类，实现抽象方法

满足以上三点，就是最基础的状态模式
